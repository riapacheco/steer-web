Introduction

What does it do?

Steer is a tool for business and team management. It simplifies work by making business goals (at all levels) easier to understand and thus easier to achieve. This allows you to focus your time on results and pull your team away from distraction.

You can build goals at a high "priority management" level for a full top-down approach, or you can choose to include goals at the project level to practically manage your commitments, costs, and time.

Profile Configurations

The profile option, found in your settings menu, allows you to set the name for your workspace, change the way your name appears to other users, and add your professional title so that new team members can gain context from the user settings page.


What's in a workspace name?

Your workspace name appears in the top-left corner of your workspace and is visible to all users.

When you grant access for certain projects to guest users, they will have their own partner workspace, where they can identify the projects you shared with them under your workspace name. This makes it so a guest user can have access to multiple workspace projects.

Speed-Up Workflow Tools

In addition to the tools that specifically help you manage priorities and projects, there are a few other tools available in Steer to help with your day-to-day workflow.



Spotlight Tab

In the top right corner of your workspace, there is a tab with three buttons (, , ). This is your spotlight tab. You can access its tools from anywhere in the workspace.


Global Search

To help you quickly find the things you need, you can perform a global search that returns your results in organized categories: Priority Cards, Project Cards, Resources, Action Items.

Trigger the search from anywhere with the keyboard shortcut ctrl + space

When action items begin to pile up, you can view them from the action item inbox that appears when you click the clipboard icon ( ) on the spotlight tab.

When you click the , you'll be able to filter the list by project card, priority card, closest critical date, assignee, or status.

User Admin

User Roles

There are 3 types of users that maintain different levels of access in Steer:
Workspace Owner, Workspace User, and Workspace Guest



Role Descriptions

Workspace Owner

The user that created the workspace in the first place is the workspace owner. They act as the primary administrator of the workspace and have all administrative privileges.

Workspace User

A workspace user is any user that accepts an invitation to an existing workspace. They have the freedom to create and edit priorities and projects with limited workspace admin privileges.

Workspace Guest

A workspace guest is any user that accepts an invitation to a shared project. They access any shared projects from their own Partner Workspace, where they can toggle between different outside workspaces.

User Settings Page

Each list item contains the user's name, a status icon, their professional title, and their email; and when you click each item, it opens their individual profile window containing more administrative tools.

Adding Workspace Users

To add a user to your workspace, click the Add New button in the top-right corner. This will open a window that looks similar to your profile window.

Once you add their email and click Save Changes, they will receive an invitation email that will both register their account and invite them to your workspace.


Deleting Users

To delete an existing user, click their name in the user list (on the user settings page). Click the button in the top-right corner of their profile window and select Delete Permanently from the menu.

Guest Users

Steer allows you to share projects with users outside of your workspace, so that they can assist in successfully completing a project without ever seeing priority-level details or other project cards within the space.



Add New Guests

  1. Press the Add New button, like you would when adding a workspace user, from the User Settings page

  2. Include their email to trigger the registration process

  3. Select at least one project for them to have access to

  4. Press Save Changes

  5. Select Send invite on the following dialog window, if you're happy with your workspace name







Add Existing Guests

If a guest user has active access to one or more of your projects, you can add them to additional projects directly from the project card itself.


  1. In your project card, click the Project Team heading found on the project team right-panel infobox

  2. Begin typing the guest user's name and select their name from the dropdown menu that appears

  3. Upon confirmation, the project will be available to their in their partner workspace

Archiving Users

By archiving a user, you can re-activate them into your workspace without the process of having them register and re-accept your workspace invitation.

The are two ways to archive a user:

  1. Within the Team Members list, right-click the user and select archive user

  2. Open the user's profile window and click the button in the top-right corner.
    Select archive user from the menu

Unarchive Users

All archived users are found on the Archived User list. To access this list, switch the Archived List toggle, in the top-right corner of the User Settings page, to the ON position.

There are two ways to unarchive users from this screen:

  1. Simply click the user's name to reveal the "unarchive" prompt

  2. Right-click the user's name and select unarchive user from the menu

Creating Cards

What are cards?

In Steer, you can create two types of cards: Priority Cards and Project Cards. Both have similar features with some key differences -- most notable difference being that Project Cards come from Priority Cards.

You'll learn more about card features in Using Cards, but for now we'll walk through how to create them.

Creating Priority Cards

Priority cards are the bigger overarching projects of your business. They typically last longer than a month and require smaller projects to reach completion.

To create a priority card, click the Add a priority card button in the left navigation bar.

A window will appear, allowing you to:

  1. Name your priority card

  2. Make the priority card private

  3. Describe the priority card for your team to better understand

Private Priority Cards

When you make a priority card private, it is only visible to you. However, you may still link in assets that live in other priority cards. This means that projects, action items, and resources that exist in other priority cards, can exist as a synced instance within your private priority card.

However, any changes made to those linked in priority card assets, are still public since its privacy is inherited from its actual parent card.

Creating Project Cards

Project cards are execution-focused and can hold any required resources & tools, specific project roles, specific project success criteria, and more.

To create a project card, you must start with an existing priority card. This ensures a dedicated focus and a cleaner way of organizing operations based on the actual work. You could optionally have a priority card act as an administrative 'placeholder' for more flexibility.

  1. Within an existing priority card, press the Add Card button found on the Project Card line.

  2. Begin typing the name of your project card

  3. When finished, simply press Enter on your keyboard and continuing typing other projects

  4. When finished listing all cards, simply click outside the project card area or escape on your keyboard
